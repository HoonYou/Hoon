# Roadmap: Agent Data Analyst
### "AI를 제어하며 본질을 꿰뚫는 분석가"

단순히 도구가 내놓는 결과를 복사하는 것이 아니라, **왜(Why)**와 **어떻게(How)**에 집중하며 AI 에이전트와 협업하여 비즈니스 가치를 창출하는 에이전트 데이터 분석가로 성장하는 과정입니다.

---

## My Vision: 에이전트 데이터 분석가란?

AI가 코드를 짜고 데이터를 전처리하는 시대에 분석가의 역할은 **설계자**이자 **해석자**가 되는 것입니다.

* **Problem Solver**: 복잡한 비즈니스 요구사항을 AI가 이해할 수 있는 구체적인 태스크로 분해합니다.
* **System Architect**: 단순 일회성 분석을 넘어, 데이터 수집부터 시각화까지 자동화된 에이전트 워크플로우를 구축합니다.
* **Decision Maker**: AI의 결과물 속에서 논리적 비약을 찾아내고, 최종 의사결정에 필요한 인사이트의 '근본'을 증명합니다.

---

## Study Strategy: 근본 있는 학습 원칙

### 1. 개념의 본질 파고들기
* **언제 & 왜?**: 새로운 라이브러리나 통계 기법을 배울 때 "이 기술이 왜 탄생했는가?", "어떤 상황에서 최적의 성능을 내는가?"를 먼저 질문합니다.
* **실무 적용**: 배운 개념을 가상의 비즈니스 시나리오(예: 커머스 이탈률, 물류 최적화)에 대입하여 프로젝트 기획안을 작성합니다.

### 2. AI 기반의 확장 학습
* **AI 프롬프팅 학습**: AI에게 단순히 코드를 짜달라고 하지 않고, 분석 로직의 검증과 **엣지 케이스(Edge Case)** 를 찾아내도록 유도하며 학습 범위를 확장합니다.
* **추상화(Abstraction)**: 비슷한 과정을 여러 개 학습한 후, 공통된 분석 패턴을 추출하여 하나의 도표나 프레임워크로 도식화합니다.

### 3. 심화 및 검증
* **논문 & 도서 탐독**: 편리한 블로그 포스팅에 의존하기보다, 해당 기술의 근간이 되는 논문이나 기본서를 통해 이론적 토대를 탄탄히 합니다.

## Anti-Patterns: 내가 경계해야 할 태도 (Don'ts)

빠른 결과물이라는 '도파민'에 속아 분석가의 근육을 퇴화시키지 않겠습니다.

| 경계해야 할 태도 | 상세 내용 |
| :--- | :--- |
| **결과 중심적 사고** | AI가 도출한 그래프와 수치를 해석 없이 그대로 수용하는 행위 |
| **회피형 코딩** | 복잡한 로직이나 에러를 만났을 때 스스로 생각하기보다 즉시 AI에게 수정을 맡기는 행위 |
| **근본 없는 구현** | 내부 동작 원리를 모른 채 복사 붙여넣기로 쌓아 올린 불안정한 코드 스택 |
| **뇌의 퇴화 방치** | AI의 지능을 나의 지능으로 착각하지 않으며, 코딩적 사고 부위를 끊임없이 단련 |

## Deep Dive: 어떻게 더 깊게 공부할 것인가?

### AI와의 페어 프로그래밍(Pair Programming) 기록
단순히 결과 코드만 올리는 것이 아니라, `logs/` 폴더를 운영하여 다음 내용을 기록합니다.

1. **AI에게 던진 질문의 의도**: 특정 문제 해결을 위해 어떤 가이드를 주었는가?
2. **AI의 오류와 교정**: AI가 범한 논리적 오류를 내가 어떻게 발견하고 바로잡았는가?
3. **트러블슈팅 과정**: 이 과정 자체가 에이전트 분석가로서의 '제어 능력'을 증명하는 강력한 포트폴리오가 됩니다.
